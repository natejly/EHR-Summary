## 🔍 Overview

The Provider Documentation Summarization Quality Instrument (PDSQI-9) is a rigorously validated instrument designed to assess the quality of clinical summaries generated by Large Language Models (LLMs). As LLMs become increasingly integrated into Electronic Health Record (EHR) workflows, ensuring the accuracy and reliability of their outputs is paramount. PDSQI-9 offers a structured framework to evaluate these summaries across multiple dimensions, facilitating safer and more effective integration of LLMs into healthcare settings.

## ✨ Key Features

**Nine Evaluation Attributes**: The instrument assesses summaries based on the following attributes:
  1. *Cited*: Includes correct and relevant citations from the source material.
  2. *Accurate*: Contains only factual, correct information with no errors.
  3. *Thorough*: Covers all important patient issues and clinical details.
  4. *Useful*: Provides information that is relevant and helpful for understanding or decision-making.
  5. *Organized*: Presents information in a logical, easy-to-follow order.
  6. *Comprehensible*: Is clear and easy to understand, with no confusing parts.
  7. *Succinct*: Is brief and to the point, without unnecessary repetition.
  8. *Synthesized*: Shows understanding by connecting information and summarizing the patient’s situation effectively.
  9. *Stigmatizing*: Uses respectful, neutral language without bias or judgment.

**Validated on Real-World Data**: The instrument was tested using summaries generated from real-world EHR data across multiple specialties.

**Robust Psychometric Properties**:
  - *Inter-Rater Reliability*: ICC = 0.867 (95% CI: 0.867–0.868)
  - *Internal Consistency*: Cronbach’s α = 0.879 (95% CI: 0.867–0.891)

**Automated Evaluation Pipeline**: This repository includes a fully automated evaluation workflow that applies the PDSQI-9 rubric using an LLM-as-a-Judge. The approach enables scalable, low-cost assessment of clinical summaries at scale.  

## 📁 Repository Structure

- [PDSQI-9 Instrument](01_PDSQI_9_Instrument/): Houses the complete PDSQI-9 instrument rubric and REDCap-ready data collection instrument to upload into your REDCap instance and run.
- [LLM-as-a-Judge](02_LLM_as_a_Judge/):  Provides prompt templates and scripts for conducting automated evaluations using the PDSQI-9 instrument with an LLM-as-a-Judge.
- [Data](03_Data/): Contains processed datasets and sample summaries from expert human evaluations. This incluse the 3 deidentified health encounters at UW Health with multiple EHR notes
- [Post-Processing and Analysis](04_Post_Processing_and_Analysis/): Includes scripts for data post-processing, statistical analyses, and visualizations.
- [Summary Generation](05_Summary_Generation/): Stores configurations and prompts used for generating summaries using different LLMs.
- [Sample Size Simulations](06_Sample_Size_Simulations/): Contains Monte Carlo simulation code for sample size planning in multi-site reliability studies of the PDSQI-9. Includes scripts for both fully crossed and partially crossed rating designs, modeling ordinal ICC precision across varying reviewer and case note configurations. 


## 🚀 Usage


### Manual Evaluation using the PDSQI-9

1. Review the [📄 PDSQI-9 Rubric (PDF)](01_PDSQI_9_Instrument/Rubric.pdf) for detailed descriptions and scoring guidance for each attribute.
2. Use the [📋 REDCap-Compatible Data Collection Form](01_PDSQI_9_Instrument/REDCap_Instrument.zip) to set up your own project or import it into an existing REDCap instance.

### Automated Evaluation using the PDSQI-9 

1. Review the available options and usage instructions [here](02_LLM_as_a_Judge).

2. We provide scripts for:
    - Running a single LLM-as-a-Judge  
    - Running multiple LLMs-as-Judges  
    - Training a customized LLM-as-a-Judge
